# Personal Finance Bot

Телеграм-бот для учёта личных доходов и расходов: лимит на траты, быстрый ввод текстом, советы по тратам, графики и разбор фото чеков.

---

## Для пользователя в Telegram

### С чего начать

1. Откройте диалог с ботом и нажмите **«Старт»** или отправьте команду `/start`.
2. Прочитайте короткое приветствие — там перечислено, что бот умеет.
3. Нажмите **«Установить бюджет»** и введите число (например, сколько денег вы готовы потратить за период). Это ваш **лимит**; бот будет показывать **остаток** после каждой записи траты.

### Кнопки главного меню

| Кнопка | Что делает |
|--------|------------|
| **Установить бюджет** | Запрашивает сумму лимита и сохраняет её. |
| **Добавить расход** | Просит ввести сумму, затем предлагает выбрать **категорию** (еда, транспорт и т.д.). |
| **Добавить доход** | То же для поступления денег (зарплата, подарок и т.д.). |
| **Советы по тратам** | Собирает вашу картину трат за последние дни и присылает **короткий текст с подсказками**. Полный режим — если у владельца бота настроены облачные подсказки (см. раздел для разработчика). |
| **Отчёт в графиках** | Присылает **картинку** — столбчатую диаграмму расходов по категориям за последние дни. |
| **Отмена** | Сбрасывает текущий шаг (например, если вы передумали вводить сумму). |

### Ввод без кнопок

Можно написать одной строкой, например:

- `Такси 200`
- `обед 500`
- `кофе 120`

Бот выделит сумму и попытается угадать категорию по словам. Если в тексте есть намёк на доход (например, «зарплата 50000»), операция может записаться как **доход**.

### Фото чека

Пришлите **фото** чека. Бот ответит, что принял снимок, и попробует **сам определить сумму и категорию**. Перед записью в учёт нужно нажать **«Подтвердить»** или **«Отмена»**. Если автоматический разбор недоступен или не удался, добавьте трату текстом, как выше.

### Команды

- `/start` — приветствие и сброс промежуточных шагов.
- `/help` — то же приветствие с подсказками.

## Для разработчика и владельца бота

### Требования

- Python **3.11+** (подойдёт и новее, например 3.13).
- Учётная запись в Telegram: создайте бота через [@BotFather](https://t.me/BotFather), получите **токен**.

### Установка

```bash
cd finance_bot
python -m venv .venv
# Windows:
.venv\Scripts\activate
# Linux/macOS:
# source .venv/bin/activate

pip install -r requirements.txt
```

### Настройка окружения

1. Скопируйте `.env.example` в `.env`.
2. Заполните переменные:

| Переменная | Обязательно | Назначение |
|------------|-------------|------------|
| `TELEGRAM_BOT_TOKEN` | Да | Токен бота от BotFather. |
| `OPENAI_API_KEY` | Нет | Если указан — работают **«Советы по тратам»** и **авторазбор фото чеков**. |
| `OPENAI_MODEL` | Нет | Модель по умолчанию: `gpt-4o-mini`. |

3. Запуск:

```bash
python main.py
```

В каталоге появится файл базы данных (путь задаётся в `config.py`). Его не коммитьте в открытый репозиторий — он уже в `.gitignore`.

### Зависимости (`requirements.txt`)

- **pyTelegramBotAPI** — работа с Telegram Bot API.
- **openai** — запросы к облачной модели для советов и чеков.
- **python-dotenv** — загрузка `.env`.
- **matplotlib** — построение PNG-графиков.
- **requests** — транзитивно используется экосистемой библиотек.

## Структура проекта и описание файлов

### Корень проекта

| Файл | Назначение |
|------|------------|
| `main.py` | Точка входа: создание бота, все обработчики сообщений и кнопок, сценарии диалога (бюджет, расход, доход, фото, советы по тратам, график). |
| `config.py` | Чтение переменных окружения, пути к `.env` и файлу базы, период отчёта по умолчанию. |
| `database.py` | Создание таблиц, запись и чтение пользователей и операций, агрегаты для отчётов и блока «Советы по тратам». |
| `keyboards.py` | Тексты кнопок и разметка **Reply**- и **Inline**-клавиатур (главное меню, категории, подтверждение чека). |
| `utils.py` | Разбор свободного текста на сумму и категорию, форматирование денег, генерация изображения графика. |
| `gpt_service.py` | Вызовы облачной модели: текстовый совет по статистике и извлечение данных с фото чека. |
| `requirements.txt` | Версии Python-пакетов. |
| `.env.example` | Шаблон секретов без реальных значений. |
| `.gitignore` | Исключения для venv, `.env`, файла БД и служебных каталогов. |
| `plan.md` | Исходное задание/план (если есть). |

## Описание модулей и функций

### `config.py`

- **`ROOT_DIR`** — каталог проекта (рядом с `config.py`).
- **`load_dotenv`** — подхватывает `.env` из `ROOT_DIR`.
- **`TELEGRAM_BOT_TOKEN`**, **`OPENAI_API_KEY`**, **`OPENAI_MODEL`** — строки из окружения.
- **`DATABASE_PATH`** — полный путь к файлу базы.
- **`DEFAULT_STATS_DAYS`** — сколько дней учитывать в сводке для графика и «Советы по тратам» (по умолчанию 7).

### `database.py`

- **`_utc_now_iso()`** — текущее время в UTC в формате ISO для полей `created_at`.
- **`get_connection()`** — контекстный менеджер соединения с БД, включение внешних ключей, commit/rollback.
- **`init_db()`** — создаёт таблицы `users` и `transactions` и индекс, если их ещё нет.
- **`get_or_create_user(user_id)`** — гарантирует строку пользователя.
- **`set_budget(user_id, budget, currency)`** — сохраняет лимит и валюту отображения.
- **`add_transaction(...)`** — добавляет доход или расход с категорией, опциональной заметкой и временем.
- **`get_user_row(user_id)`** — одна строка пользователя или `None`.
- **`sum_expenses_all_time(user_id)`** — сумма всех расходов пользователя (для остатка от лимита).
- **`sum_for_period(user_id, tx_type, days)`** — сумма доходов или расходов за последние `days` дней.
- **`get_expenses_by_category(user_id, days)`** — словарь «категория → сумма» за период.
- **`get_recent_transactions(user_id, limit)`** — последние операции для контекста совета по тратам.
- **`get_balance_snapshot(user_id, period_days)`** — сводка: бюджет, валюта, расходы/доходы за период, разбивка по категориям.
- **`_parse_iso(ts)`** — проверка/разбор ISO-времени при явной передаче `created_at` в транзакцию.

### `keyboards.py`

Константы **`BTN_*`** — точные подписи кнопок; по ним в `main.py` распознаётся нажатие.

- **`main_reply_keyboard()`** — основное меню под полем ввода.
- **`cancel_reply_keyboard()`** — одна кнопка «Отмена» на шагах ввода.
- **`EXPENSE_CATEGORIES`**, **`INCOME_CATEGORIES`** — внутренний ключ → подпись на русском.
- **`category_inline_markup(tx_kind, prefix)`** — inline-кнопки категорий; `callback_data` вида `pick:expense:food`.
- **`receipt_confirm_markup()`** — «Подтвердить» / «Отмена» для чека.

### `utils.py`

- **`ParsedTransaction`** — датакласс: тип операции, сумма, ключ категории, исходная строка.
- **`parse_expense_text(text)`** — извлекает сумму и категорию из фразы пользователя.
- **`_category_from_label`**, **`_first_amount_in_text`**, **`_normalize_amount`** — вспомогательная логика парсинга.
- **`format_money(amount, currency)`** — единообразный вывод суммы с валютой.
- **`build_spending_chart_png(user_id, days, expenses_fetcher)`** — строит PNG в памяти (`BytesIO`); при отсутствии данных — заглушка с текстом.
- **`match_amount_pattern(text)`** — проверка текста регулярным выражением (используется в `main` для отладочного лога).

### `gpt_service.py`

- **`_get_client()`** — ленивая инициализация клиента API (таймаут, число повторов).
- **`_category_ru(cat_key, tx_type)`** — человекочитаемая категория для текста, отправляемого модели.
- **`get_financial_advice(user_data)`** — формирует текст статистики, запрашивает у модели короткий совет на русском; при отсутствии ключа или ошибке возвращает **понятное пользователю** сообщение без технических деталей.
- **`parse_receipt_image(image_bytes, mime)`** — возвращает словарь с суммой, ключом категории и опционально названием магазина, либо `None`.

### `main.py`

Состояние диалога хранится в словарях **`user_fsm`** (шаг сценария, временная сумма) и **`pending_receipt`** (данные с чека до подтверждения).

- **`_bot()`** — создаёт экземпляр `TeleBot`; при отсутствии токена завершает процесс.
- **`_parse_amount(text)`** — выдёргивает положительное число из строки.
- **`_welcome_text()`** — текст приветствия для пользователя (без технического жаргона).
- **`_remainder_line(user_id)`** — строка с остатком после операции или подсказка задать лимит.
- **`cmd_start`**, **`cmd_help`** — команды `/start` и `/help`.
- **`on_photo`** — загрузка файла, вызов разбора чека, показ подтверждения.
- **`on_receipt_callback`**, **`on_pick_category`** — обработка нажатий inline-кнопок (чек и категории).
- **`on_text`** — маршрутизация: кнопки меню, шаги FSM (бюджет, сумма, выбор категории), свободный текст, fallback с подсказкой.
- **`main()`** — `init_db()`, запуск бесконечного опроса Telegram.

## Поток данных (упрощённо)

1. Пользователь нажимает кнопку или пишет текст → **`main.py`**.
2. Нужна запись в учёт → **`database.py`**.
3. Нужен разбор фразы → **`utils.py`**.
4. Нужен график → **`utils.build_spending_chart_png`** → картинка в чат.
5. Нужен совет или чек по фото → **`gpt_service.py`** (если в `.env` задан ключ), результат снова через **`main.py`** в Telegram.
